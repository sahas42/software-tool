# AspireBridge API Endpoints Reference

Complete API documentation for all backend endpoints.

## Base URL
```
http://localhost:3001/api
```

---

## Authentication Endpoints

### POST /auth/register
Create a new user account.
- **Auth Required:** No
- **Body:** `{ username, email, password }`
- **Response:** `{ user: UserPublic }`

### POST /auth/login
Authenticate a user.
- **Auth Required:** No
- **Body:** `{ username, password }`
- **Response:** `{ user: UserPublic }`

### POST /auth/logout
Log out current user.
- **Auth Required:** Yes
- **Response:** `{ message }`

### GET /auth/me
Get current user information.
- **Auth Required:** Yes
- **Response:** `{ user: UserPublic }`

---

## Book Listing Endpoints

### GET /books
Get all book listings (public, with optional filters).
- **Auth Required:** No
- **Query Params:** `exam_type`, `location_city`, `location_state`, `status`
- **Response:** `BookListing[]`

### GET /books/:id
Get single book listing.
- **Auth Required:** No
- **Response:** `BookListing`

### POST /books
Create new book listing.
- **Auth Required:** Yes (Donor)
- **Body:** `{ title, description, exam_type, subjects[], condition, year_used, location_city, location_state }`
- **Response:** `BookListing`

### PUT /books/:id
Update book listing.
- **Auth Required:** Yes (Owner only)
- **Body:** `{ title?, description?, condition?, status? }`
- **Response:** `BookListing`

### DELETE /books/:id
Delete book listing.
- **Auth Required:** Yes (Owner only)
- **Response:** `{ message }`

### GET /my-listings
Get logged-in donor's listings.
- **Auth Required:** Yes
- **Response:** `BookListing[]`

### POST /books/:id/images
Upload images for a book listing.
- **Auth Required:** Yes (Owner only)
- **Body:** FormData with `images` field (up to 5 images, 5MB each)
- **Response:** `{ images: string[] }`

## Recipient Profile Endpoints

### POST /recipient/profile
Create recipient profile.
- **Auth Required:** Yes
- **Body:** `{ full_name, phone_number, date_of_birth, current_class, target_exam, tenth_percentage, tenth_board, school_name, school_address, school_city, school_state, school_pincode, school_id? }`
- **Response:** `RecipientProfile`

### GET /recipient/profile
Get own recipient profile.
- **Auth Required:** Yes
- **Response:** `RecipientProfile`

### PUT /recipient/profile
Update recipient profile.
- **Auth Required:** Yes
- **Body:** Partial `RecipientProfile` fields
- **Response:** `RecipientProfile`

### POST /recipient/marksheet
Upload 10th marksheet.
- **Auth Required:** Yes
- **Body:** FormData with `marksheet` field (PDF/JPEG/PNG, max 10MB)
- **Response:** `{ marksheet_file: string }`

---

## Book Request Endpoints

### POST /requests
Create book request (verified recipient only).
- **Auth Required:** Yes (Verified recipient)
- **Body:** `{ listing_id }`
- **Response:** `BookRequest`
- **Notes:** Automatically calculates priority score

### GET /requests/my-requests
Get user's book requests.
- **Auth Required:** Yes
- **Response:** `BookRequest[]`

### GET /requests/listing/:listingId
Get requests for a specific listing.
- **Auth Required:** Yes (Donor/Admin only)
- **Response:** `BookRequest[]`

### DELETE /requests/:id
Cancel a book request.
- **Auth Required:** Yes (Owner only)
- **Response:** `{ message }`

---

## School Endpoints

### GET /schools/search
Search schools by name or city.
- **Auth Required:** No
- **Query Params:** `q` (search query)
- **Response:** `School[]` (max 50 results)

### POST /schools
Add new school (or return existing).
- **Auth Required:** Yes
- **Body:** `{ name, address, city, state, pincode, board? }`
- **Response:** `School`

---

## Notification Endpoints

### GET /notifications
Get user's notifications.
- **Auth Required:** Yes
- **Query Params:** `unread=true` (optional)
- **Response:** `Notification[]` (max 50)

### PUT /notifications/:id/read
Mark notification as read.
- **Auth Required:** Yes
- **Response:** `{ message }`

### PUT /notifications/read-all
Mark all notifications as read.
- **Auth Required:** Yes
- **Response:** `{ message }`

### GET /notifications/unread-count
Get unread notification count.
- **Auth Required:** Yes
- **Response:** `{ count: number }`

---

## Admin Endpoints

### GET /admin/profiles/pending
Get unverified recipient profiles.
- **Auth Required:** Yes (Admin only)
- **Response:** `RecipientProfile[]`

### POST /admin/profiles/:id/verify
Approve or reject a recipient profile.
- **Auth Required:** Yes (Admin only)
- **Body:** `{ approved: boolean, reason?: string }`
- **Response:** `{ message }`
- **Side Effects:** Creates notification for recipient

### GET /admin/requests
Get all pending book requests.
- **Auth Required:** Yes (Admin only)
- **Response:** `BookRequest[]`

### POST /admin/requests/:id/approve
Approve a book request and allocate listing.
- **Auth Required:** Yes (Admin only)
- **Body:** `{ notes?: string }`
- **Response:** `{ message }`
- **Side Effects:** 
  - Allocates listing to recipient
  - Rejects other requests for same listing
  - Creates notification for recipient

### POST /admin/requests/:id/reject
Reject a book request.
- **Auth Required:** Yes (Admin only)
- **Body:** `{ reason: string }`
- **Response:** `{ message }`
- **Side Effects:** Creates notification for recipient

### GET /admin/dashboard
Get dashboard statistics.
- **Auth Required:** Yes (Admin only)
- **Response:** `{ totalListings, availableListings, pendingVerifications, pendingRequests }`

## Error Responses

All endpoints return standard error format:
```json
{
  "error": "Error message"
}
```

Common status codes:
- `400` - Bad Request (validation error)
- `401` - Unauthorized (not logged in)
- `403` - Forbidden (insufficient permissions)
- `404` - Not Found
- `500` - Internal Server Error

---

## Notes

- All authenticated endpoints require session cookie
- Frontend must include `credentials: 'include'` in fetch requests
- File uploads use `multipart/form-data`
- JSON endpoints expect `Content-Type: application/json`
- Priority score calculation: `(merit × 0.7) + proximity` where merit is 10th percentage and proximity is 0-30 based on location match
