# Implementation Plan: Login & User Management for Donation App

## Overview
Add complete authentication, user management, and authorization to the donation app. Users will be able to register, login, manage their profiles, and only delete their own donations.

## Current State (donation-app-02-sqlite)
- Full-stack TypeScript app (Express backend + React frontend)
- SQLite database with single `donations` table (id, name, email, amount, timestamp)
- No authentication or user management
- Anyone can view/create/delete any donation

## Target State (donation-app-03-login)
- User registration and login with password authentication
- Session-based authentication with HttpOnly cookies
- Settings page: update profile, change password, upload profile picture, delete account
- Users can only delete their own donations
- Database with users table (role: admin/client, default: client)
- Admin functionality ready for next iteration

---

## Architecture Decisions

### 1. **Authentication: Session-Based (not JWT)**
- **Why:** Simpler for same-origin apps, more secure (HttpOnly cookies), easier to revoke
- **Tech:** express-session + better-sqlite3-session-store + bcrypt (12 rounds)
- **Session Storage:** SQLite (survives restarts, no memory limitations)

### 2. **Profile Pictures: Filesystem (not database)**
- **Why:** Better performance, standard practice, easy to cache
- **Storage:** `backend/uploads/profiles/user_{id}.{ext}`
- **Serving:** Express static middleware
- **Upload:** multer middleware (max 2MB, JPEG/PNG only)

### 3. **Frontend Routing: React Router**
- **Routes:** `/` (donations), `/login`, `/register`, `/settings`
- **Protection:** ProtectedRoute component for authenticated-only pages

### 4. **State Management: Context API**
- **AuthContext:** Global auth state (user, login, logout, register)
- **Why:** Built-in, perfect for single global state, simpler than Redux

## Database Schema Changes

### New Tables

**users table:**
```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  username TEXT NOT NULL UNIQUE,
  email TEXT NOT NULL UNIQUE,
  password_hash TEXT NOT NULL,
  role TEXT NOT NULL DEFAULT 'client',  -- 'admin' or 'client'
  profile_picture TEXT,                  -- filename (e.g., "user_5.jpg")
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);
```

**sessions table:**
```sql
CREATE TABLE sessions (
  sid TEXT PRIMARY KEY,
  sess TEXT NOT NULL,
  expired INTEGER NOT NULL
);
```

### Modified Table

**donations table:**
```sql
-- Add user_id column (nullable for existing donations)
ALTER TABLE donations ADD COLUMN user_id INTEGER REFERENCES users(id) ON DELETE SET NULL;
```

**Migration Strategy:** Existing donations keep `user_id = NULL` (anonymous/legacy donations)

## Backend Changes

### New Dependencies
```json
{
  "express-session": "^1.18.1",
  "better-sqlite3-session-store": "^1.0.0",
  "bcrypt": "^5.1.1",
  "multer": "^1.4.5-lts.1"
}
```

### New Files

**backend/src/middleware/auth.ts**
- `requireAuth()` - Middleware to protect routes requiring login
- `requireRole(role)` - Middleware for admin-only routes (future use)

**backend/src/validation/auth.ts**
- `validateRegistration()` - Username, email, password validation
- Password requirements: 8+ chars, uppercase, lowercase, number

**backend/src/types/express-session.d.ts**
- TypeScript session interface extension (userId, username, role)

### Modified Files

**backend/src/database.ts** - Add user management functions:
- `createUser(username, email, password)` - Register new user
- `getUserByUsername(username)` - For login
- `getUserById(id)` - For session validation
- `updateUserProfile(userId, updates)` - Edit profile
- `updateUserPassword(userId, newPassword)` - Change password
- `updateProfilePicture(userId, filename)` - Save profile pic filename
- `deleteUser(userId)` - Delete account
- Modify `addDonation()` to accept optional `userId`
- Modify `deleteDonation(id, userId)` to check ownership

**backend/src/server.ts** - Add authentication endpoints:

**Auth endpoints:**
- `POST /api/auth/register` - Create account
- `POST /api/auth/login` - Login
- `POST /api/auth/logout` - Logout
- `GET /api/auth/me` - Get current user (check if logged in)

**Settings endpoints (protected):**
- `PUT /api/user/profile` - Update username/email
- `PUT /api/user/password` - Change password
- `POST /api/user/profile-picture` - Upload profile picture (multer)
- `DELETE /api/user/account` - Delete account

**Modified endpoints:**
- `POST /api/donations` - Auto-associate with logged-in user
- `DELETE /api/donations/:id` - Check ownership before deleting

**Static serving:**
- `app.use('/api/uploads', express.static('uploads'))` - Serve profile pictures

**Session configuration:**
```typescript
app.use(session({
  store: new SqliteStore({ client: db }),
  secret: process.env.SESSION_SECRET,
  resave: false,
  saveUninitialized: false,
  cookie: {
    httpOnly: true,
    sameSite: 'lax',
    maxAge: 24 * 60 * 60 * 1000  // 24 hours
  }
}));
```

**CORS configuration:**
```typescript
app.use(cors({
  origin: 'http://localhost:5173',
  credentials: true  // Critical for cookies!
}));
```

## Security Features

✅ **Password Hashing:** bcrypt with 12 salt rounds
✅ **SQL Injection Prevention:** Parameterized queries
✅ **XSS Prevention:** React escaping + HttpOnly cookies
✅ **CSRF Protection:** SameSite cookie attribute
✅ **Input Validation:** Client-side and server-side
✅ **File Upload Security:** Type/size restrictions
✅ **Authorization:** Ownership checks before deletion
✅ **Environment Variables:** SESSION_SECRET in .env (gitignored)
