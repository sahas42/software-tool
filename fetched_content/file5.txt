## Executive Summary

AspireBridge is a comprehensive web-based platform designed to connect book donors with students in need of educational materials. The platform facilitates the donation, verification, and allocation of books through a multi-role system with administrative oversight. The application features intelligent priority-based allocation, real-time notifications, and a responsive user interface optimized for both desktop and mobile devices.

**Key Metrics**:
- 3 User Roles (Donor, Recipient, Admin)
- 5 Database Tables
- 25+ API Endpoints
- 15+ Frontend Pages/Components
- Responsive Design (Mobile & Desktop)

## Feature Overview

### 1. Multi-Role Authentication System

**Description**: A comprehensive authentication system supporting multiple user roles with session-based security.

**Key Components**:
- User registration with role selection (Donor, Recipient, or Both)
- Secure login with bcrypt password hashing (12 salt rounds)
- Session management using SQLite-backed sessions
- HttpOnly cookies for CSRF protection
- Automatic session persistence across server restarts

**User Roles**:
- **Donor**: Can create and manage book listings
- **Recipient**: Can browse books and submit requests
- **Both**: Combined donor and recipient capabilities
- **Admin**: Full platform oversight and approval authority

**Technical Implementation**:
- `express-session` with `better-sqlite3-session-store`
- React Context API for frontend state management
- Protected routes with role-based access control
- Guest route protection (prevents logged-in users from accessing login/register)

### 4. Intelligent Book Allocation System

**Description**: Priority-based allocation algorithm ensuring fair distribution of educational resources.

**Priority Scoring Algorithm**:
```
Total Score = (Academic Merit × 0.7) + (Proximity × 0.3)
```

**Academic Merit (70% weight)**:
- 90%+ marks: Score 1.0
- 80-89% marks: Score 0.8
- 70-79% marks: Score 0.6
- 60-69% marks: Score 0.4
- Below 60%: Score 0.2

**Proximity (30% weight)**:
- Same city as donor: Score 1.0
- Same state as donor: Score 0.5
- Different state: Score 0.0

**Features**:
- Automatic priority calculation on request submission
- Admin visibility of priority scores
- Fair distribution based on merit and location
- Prevents duplicate requests for same book

**Technical Implementation**:
- `allocation.ts` module with scoring logic
- Priority score stored in `book_requests` table
- Admin dashboard displays sorted requests
- Automatic status updates on approval

- Foreign keys with CASCADE/SET NULL
- Indexed columns for performance
- Unique constraints for data integrity

## Security Implementation

### Authentication Security
- **Password Hashing**: Bcrypt with 12 salt rounds
- **Session Management**: Secure session cookies
- **HttpOnly Cookies**: Prevents XSS attacks
- **SameSite Attribute**: CSRF protection

### Authorization
- **Role-Based Access**: Admin, donor, recipient roles
- **Ownership Checks**: Users can only modify their own data
- **Protected Routes**: Frontend and backend protection
- **Admin-Only Endpoints**: Restricted administrative functions

### Input Validation
- **Client-Side**: Form validation in React
- **Server-Side**: Express validation middleware
- **SQL Injection Prevention**: Parameterized queries
- **File Upload Security**: Type and size restrictions

### Data Protection
- **Foreign Keys**: Referential integrity
- **Cascade Rules**: Proper data cleanup
- **Status Validation**: Prevents invalid state changes
- **Transaction Safety**: Atomic operations
