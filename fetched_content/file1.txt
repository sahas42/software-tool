# AspireBridge - Educational Book Donation Platform

A full-stack web application connecting book donors with students in need. AspireBridge facilitates the donation and allocation of educational materials to verified students, featuring role-based access, an admin approval system, and intelligent book allocation based on academic merit and proximity.

## 🚀 Quick Start

```bash
# 1. Navigate to backend directory and install dependencies
cd backend
npm install

# 2. Create admin user (first time only)
npm run create-admin

# 3. Start the backend server (runs on http://localhost:3001)
npm run dev

# 4. In a new terminal, navigate to frontend directory
cd ../frontend
npm install

# 5. Start the frontend development server (runs on http://localhost:5173)
npm run dev

# 6. Open your browser to http://localhost:5173
```

## ✨ Features

### 🎯 Core Functionality

#### For Donors
- **Create Book Listings**: Post books with title, author, subject, condition, and images
- **Manage Listings**: View and track all your book donations
- **Allocation Notifications**: Get notified when your books are allocated to students
- **Delete Protection**: Cannot delete books that have been allocated

#### For Recipients (Students)
- **Browse Books**: Search and filter available educational books
- **Student Profile**: Submit academic information and school details for verification
- **Request Books**: Request books that match your educational needs
- **Track Requests**: Monitor the status of all your book requests
- **Priority Scoring**: Requests ranked by academic merit (70%) and proximity (30%)

#### For Administrators
- **Admin Dashboard**: Overview of platform statistics and pending actions
- **Profile Verification**: Review and approve/reject student profiles
- **Request Management**: Approve or reject book allocation requests
- **Priority-Based Allocation**: View calculated priority scores for fair distribution

### 🔐 Authentication & Authorization
- **Multi-Role System**: Users can be donors, recipients, or both
- **Session-Based Auth**: Secure HttpOnly cookies with SQLite session storage
- **Protected Routes**: Role-based access control for different features
- **Guest Route Protection**: Prevents logged-in users from accessing login/register pages
- **Redirect After Login**: Returns users to their intended destination

### 📱 User Interface
- **Responsive Sidebar Navigation**: Clean, organized navigation that persists across pages
- **Mobile-Friendly**: Fully responsive design for all screen sizes
- **Role-Based Menus**: Navigation adapts based on user roles
- **Hamburger Menu**: Toggle sidebar from left side of navbar
- **Notifications Center**: Real-time updates on requests and allocations

### 🔔 Notification System
- **Request Status Updates**: Recipients notified of approval/rejection
- **Allocation Alerts**: Donors notified when books are allocated
- **Admin Notes**: Rejection reasons and feedback
- **Unread Tracking**: Mark notifications as read/unread

### 🎓 Student Verification
- **Academic Information**: 10th grade percentage, current class
- **School Details**: School name, city, state
- **Admin Review**: Manual verification by administrators
- **Status Tracking**: Pending, approved, or rejected states

### 📊 Intelligent Allocation
- **Priority Algorithm**: Merit-based scoring system
  - Academic Merit: 70% weight (based on 10th grade %)
  - Proximity: 30% weight (same city > same state > different state)
- **Fair Distribution**: Ensures books reach deserving students
- **Admin Oversight**: Final approval by administrators

## 🛠️ Tech Stack

### Backend
- **Runtime**: Node.js with TypeScript
- **Framework**: Express.js 5.2.1
- **Database**: SQLite with better-sqlite3
- **Authentication**: express-session with SQLite session store
- **Password Security**: bcrypt (12 salt rounds)
- **File Upload**: multer (for book images and profile pictures)
- **CORS**: Configured for frontend communication

### Frontend
- **Framework**: React 19.2.0 with TypeScript
- **Build Tool**: Vite 7.2.4
- **Routing**: react-router-dom 7.6.3
- **State Management**: React Context API
- **Styling**: CSS with responsive design

## 🗄️ Database Schema

### users
```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  username TEXT NOT NULL UNIQUE,
  email TEXT NOT NULL UNIQUE,
  password_hash TEXT NOT NULL,
  role TEXT NOT NULL DEFAULT 'user',      -- 'admin' or 'user'
  user_type TEXT,                         -- 'donor', 'recipient', or 'both'
  profile_picture TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);
```

### book_listings
```sql
CREATE TABLE book_listings (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  donor_id INTEGER NOT NULL,
  title TEXT NOT NULL,
  author TEXT NOT NULL,
  subject TEXT NOT NULL,
  book_condition TEXT NOT NULL,
  description TEXT,
  status TEXT NOT NULL DEFAULT 'available',  -- 'available' or 'allocated'
  city TEXT NOT NULL,
  state TEXT NOT NULL,
  images TEXT,                               -- JSON array of filenames
  created_at TEXT NOT NULL,
  FOREIGN KEY (donor_id) REFERENCES users(id) ON DELETE CASCADE
);
```

### recipient_profiles
```sql
CREATE TABLE recipient_profiles (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL UNIQUE,
  tenth_percentage REAL NOT NULL,
  current_class INTEGER NOT NULL,
  school_name TEXT NOT NULL,
  city TEXT NOT NULL,
  state TEXT NOT NULL,
  verification_status TEXT NOT NULL DEFAULT 'pending',  -- 'pending', 'approved', 'rejected'
  admin_notes TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

### book_requests
```sql
CREATE TABLE book_requests (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  book_id INTEGER NOT NULL,
  recipient_id INTEGER NOT NULL,
  status TEXT NOT NULL DEFAULT 'pending',  -- 'pending', 'approved', 'rejected'
  priority_score REAL,
  admin_notes TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (book_id) REFERENCES book_listings(id) ON DELETE CASCADE,
  FOREIGN KEY (recipient_id) REFERENCES users(id) ON DELETE CASCADE
);
```

### notifications
```sql
CREATE TABLE notifications (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  type TEXT NOT NULL,                      -- 'profile_approved', 'request_approved', etc.
  title TEXT NOT NULL,
  message TEXT NOT NULL,
  link TEXT,
  is_read INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

## 🔌 API Endpoints

### Authentication
- `POST /api/auth/register` - Create new user account
- `POST /api/auth/login` - Authenticate user
- `POST /api/auth/logout` - End session
- `GET /api/auth/me` - Get current user info

### Book Listings
- `GET /api/books` - Get all available books (with filters)
- `GET /api/books/:id` - Get single book details
- `POST /api/books` - Create book listing (donor only)
- `DELETE /api/books/:id` - Delete listing (donor only, not if allocated)
- `GET /api/my-listings` - Get logged-in donor's listings

### Recipient Profiles
- `GET /api/recipient-profile` - Get own profile
- `POST /api/recipient-profile` - Create/update profile
- `GET /api/admin/recipient-profiles/pending` - Get pending verifications (admin)
- `PUT /api/admin/recipient-profiles/:id/verify` - Approve/reject profile (admin)

### Book Requests
- `POST /api/books/:id/request` - Request a book (recipient only)
- `GET /api/my-requests` - Get own requests
- `GET /api/admin/book-requests/pending` - Get pending requests (admin)
- `PUT /api/admin/book-requests/:id` - Approve/reject request (admin)

### Notifications
- `GET /api/notifications` - Get user's notifications
- `PUT /api/notifications/:id/read` - Mark as read
- `PUT /api/notifications/mark-all-read` - Mark all as read

### Admin
- `GET /api/admin/stats` - Get dashboard statistics

### User Settings
- `PUT /api/user/profile` - Update username/email
- `PUT /api/user/password` - Change password
- `POST /api/user/profile-picture` - Upload profile picture
- `DELETE /api/user/account` - Delete account

## 👥 User Roles & Workflows

### Donor Workflow
1. Register/Login as a user
2. Select "Donor" as user type during registration
3. Create book listings with details and images
4. View and manage listings in "My Listings"
5. Receive notifications when books are allocated
6. Cannot delete allocated books

### Recipient Workflow
1. Register/Login as a user
2. Select "Recipient" as user type during registration
3. Complete student profile with academic information
4. Wait for admin verification (pending → approved)
5. Browse available books
6. Request books that match educational needs
7. Track request status in "My Requests"
8. Receive notifications on request approval/rejection

### Admin Workflow
1. Login with admin credentials
2. Access Admin Dashboard for overview
3. Review pending student profiles in "Verifications"
4. Approve/reject profiles with optional notes
5. Review book requests in "Requests"
6. View priority scores for fair allocation
7. Approve/reject requests with optional notes
8. System automatically sends notifications to users

## 🐛 Troubleshooting

### Common Issues

**Sessions not persisting**
- Check `SESSION_SECRET` is set in `.env`
- Verify sessions table exists in database
- Ensure `credentials: 'include'` in all fetch requests

**CORS errors**
- Verify frontend URL matches backend CORS config
- Check `credentials: true` in CORS settings
- Ensure cookies are being sent (check DevTools)

**File uploads failing**
- Check file size (max 5MB for books, 2MB for profiles)
- Verify file types (JPEG/PNG only)
- Ensure upload directories exist

**Admin panel not accessible**
- Verify user has `role='admin'` in database
- Check admin user was created with `npm run create-admin`
- Clear browser cache and cookies
